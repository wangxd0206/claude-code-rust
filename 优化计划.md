## 🔴 高优先级（影响核心体验）

### 1. API 流式传输
**问题**：
- REPL 和 query 都使用 `stream: false`
- 等待完整响应才输出，用户体验差
- API 模块有 `chat_stream` 方法但未使用

**位置**：
- `src/cli/repl.rs:96` - `stream: false`
- `src/cli/args.rs:119` - `stream: false`

**实现方案**：
1. 修改 REPL 使用异步流式处理
2. 解析 SSE（Server-Sent Events）格式
3. 实时输出 token，营造打字机效果
4. 保持完整消息历史记录

**预期收益**：
- 输出速度提升 80%+
- 用户可实时看到响应
- 体验与原版一致

---

### 2. MCP 工具调用集成
**问题**：
- 有完整的 MCP Manager 模块
- 但 REPL 对话流程中无工具调用
- 无法使用文件操作、命令执行等工具

**位置**：
- `src/mcp/` - MCP 模块已实现
- `src/cli/repl.rs` - 需集成工具调用

**实现方案**：
1. 在 API 请求中添加 tools 参数
2. 处理 tool_calls 响应
3. 执行对应的 MCP 工具
4. 将工具结果返回给 API
5. 获取最终响应

**预期收益**：
- 可使用所有原版 Claude 的工具
- 文件操作、命令执行等
- 完整的 AI 编程助手体验
